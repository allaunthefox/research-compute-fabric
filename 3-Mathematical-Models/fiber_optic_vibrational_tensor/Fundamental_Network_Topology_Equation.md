## Fundamental Equation

### Primary Network Efficiency Equation

```
E(N) = (Σᵢ wᵢ · αᵢ · fᵢ(N)) · P(N) · I(N) · S(N) · (1 - λ·C(N))
```

Where:
- **E(N)**: Overall network efficiency for topology N
- **wᵢ**: Weight of methodology i (based on current evidence/alignment strength)
- **αᵢ**: Alignment score of methodology i with soliton analysis
- **fᵢ(N)**: Normalized score from methodology i for topology N
- **P(N)**: Physics constraint factor (latency, power, distance)
- **I(N)**: Infrastructure density factor
- **S(N)**: Strategic importance factor
- **C(N)**: Normalized excess complexity and hidden-route cost
- **λ**: Simplicity coefficient (empirically ~0.3)

### Physics Constraint Factor

```
P(N) = exp(-β·(L_avg/L_min) · (P_total/P_max) · (D_geo/D_optimal))
```

Where:
- **L_avg**: Average latency in network
- **L_min**: Minimum achievable latency (physics limit)
- **P_total**: Total power consumption
- **P_max**: Maximum available power
- **D_geo**: Geographic dispersion
- **D_optimal**: Optimal geographic dispersion
- **β**: Physics constraint coefficient (~0.5)

### Infrastructure Density Factor

```
I(N) = 1 - exp(-γ·(D_actual/D_optimal))
```

Where:
- **D_actual**: Actual infrastructure density
- **D_optimal**: Optimal infrastructure density
- **γ**: Density coefficient (~0.8)

### Strategic Importance Factor

```
S(N) = Σⱼ sⱼ · wⱼ
```

Where:
- **sⱼ**: Strategic score of node j
- **wⱼ**: Weight of node j (based on power consumption, connectivity, etc.)

### Network Complexity Measure

```
C(N) = clamp(((E_actual/E_theoretical) · max(0, R_actual - R_optimal)/R_budget), 0, 1/λ)
```

Where:
- **E_actual**: Actual number of edges
- **E_theoretical**: Theoretical minimum edges
- **R_actual**: Actual redundancy
- **R_optimal**: Optimal useful redundancy
- **R_budget**: Redundancy budget used to normalize excess redundancy

Beneficial redundancy belongs in `S(N)` or reliability-aware route scoring.
`C(N)` only penalizes excess edges, hidden-route cost, and anti-good-enough
complexity after normalization. The clamp keeps `(1 - λ·C(N))` from becoming
negative or rewarding under-defined complexity.
